## 엔진의 구조: 7계층

```
┌──────────────────────────────────────┐
│ Layer 0: Curiosity (입력)            │
│ 예: "AI", "물", "조선", "맥도날드" │
└────────────┬─────────────────────────┘
             │
┌────────────▼─────────────────────────┐
│ Layer 1: Question (질문 생성)       │
│ "AI의 병목은?"                      │
│ "물 산업의 불균형은?"              │
└────────────┬─────────────────────────┘
             │
             ↓ Question이 다음을 결정
             │
┌────────────▼─────────────────────────┐
│ Layer 2: Evidence (증거 수집)        │
│ Question 기반 필요 데이터만 수집    │
└────────────┬─────────────────────────┘
             │
┌────────────▼─────────────────────────┐
│ Layer 3: Statistics (통계 계산)     │
│ 수집한 증거에서 필요한 수치만 추출  │
└────────────┬─────────────────────────┘
             │
┌────────────▼─────────────────────────┐
│ Layer 4: Pattern (패턴 식별)         │
│ 통계의 시간에 따른 변화             │
└────────────┬─────────────────────────┘
             │
┌────────────▼─────────────────────────┐
│ Layer 5: Observation (관찰)         │
│ 패턴을 설명하는 추가 증거           │
│ Confidence 지표 추가                │
└────────────┬─────────────────────────┘
             │
┌────────────▼─────────────────────────┐
│ Layer 6: Hypothesis (임시 결론)     │
│ "다음 병목은..."                    │
│ Confidence 명시                     │
└──────────────────────────────────────┘
```

## 예시: "AI" 입력

```
Curiosity
├─ 입력: "AI"
│
Question (자동 생성)
├─ "AI 산업의 현재 병목은?"
├─ "다음 제약요소는?"
├─ "자본이 모이는 곳은?"
│
Evidence (Question 기반)
├─ AI/GPU/HBM/Power/Cooling 기사
├─ 펀딩 정보
├─ 공시 자료
├─ 통계 데이터
│
Statistics (필요한 것만)
├─ GPU 기사: 월별 +분야별
├─ HBM 기사: 월별
├─ Power 기사: 월별
├─ Cooling 기사: 월별
├─ 펀딩 금액: 분기별
├─ 가격 변동: 일일
│
Pattern (시간에 따른 변화)
├─ GPU: 높음 → 안정
├─ HBM: 낮음 → 높음 → 계속 높음
├─ Power: 매우낮음 → 증가 (최근)
├─ Cooling: 낮음 (변화 없음)
│
Observation (패턴 설명)
├─ HBM이 왜 계속 증가?
│   → SK하이닉스, Samsung 투자
│   → 정부 정책 (반도체 육성)
│   → 데이터센터 계약 증가
│
├─ Power가 최근 증가?
│   → AI 데이터센터 전력 부하
│   → 냉각 수요 증가
│   → 전력 인프라 계약 발표 증가
│
Hypothesis (최종 가설)
├─ "AI 산업의 현재 병목: HBM"
│   Confidence: ★★★★★ 87%
│   Evidence: 686개 기사 | Sources: 124개 | Period: 30일
│
├─ "다음 병목: Power/에너지"
│   Confidence: ★★★☆☆ 43%
│   Evidence: 2개 기사 (간접) | Sources: 0 | Period: 30일
│   Counter Evidence: 관찰 기간 짧음, 독립 출처 없음
│   Need: 더 많은 직접 증거 필요
│
└─ "병목 해결처: 원전·배터리"
    Confidence: ★★★☆☆ 41%
```

```
Question: "AI 산업의 병목은?"
└─ Statistics 항목:
   ├─ GPU 뉴스 (✓ 수집)
   ├─ HBM 뉴스 (✓ 수집)
   ├─ Power 뉴스 (✓ 수집)
   ├─ Cooling 뉴스 (✓ 수집)
   └─ 날씨 데이터 (✗ 무관)

Question: "가뭄이 물 시장에 미치는 영향?"
└─ Statistics 항목:
   ├─ 강수량 (✓ 수집)
   ├─ 댐 수위 (✓ 수집)
   ├─ 용수 가격 (✓ 수집)
   ├─ 곡물 수확 (✓ 수집)
   └─ GPU 뉴스 (✗ 무관)
```

**같은 데이터 소스도 Question에 따라 다르게 처리된다.**

## 엔진의 강점

### 1. 범용성 (Universal)
```
모든 산업, 모든 기업, 모든 상품 분석 가능
같은 로직, 다른 Curiosity
```

### 2. 투명성 (Transparent)
```
각 계층이 명확
증거 수, 신뢰도, 반박 의견 명시
재현 가능
```

### 3. 객관성 (Objective)
```
Question이 정해지면 자동으로 분석
주관성 개입 최소화
확인 가능성 최대화
```

### 4. 진화성 (Evolving)
```
더 많은 증거 → Confidence 상승
가설은 계속 수정됨
잘못된 가설도 증거로 수정됨
```

## 주의: 이 엔진이 할 수 없는 것

### ❌ 할 수 없음
```
- 절대 답 제시
- 미래 예측
- 확실한 투자 조언
- 100% 신뢰도
- "반드시"라는 표현
```

### ✅ 할 수 있음
```
- 현재 패턴 분석
- 과거 설명
- 신호 인식
- 가능성 제시
- "일 수 있다"는 표현
```

## 최종 정리

### 엔진이 하는 일
1. Curiosity 입력받기
2. Question 생성
3. Evidence 수집
4. Statistics 계산
5. Pattern 인식
6. Observation 분석
7. Hypothesis 제시

"좋은 질문을 통해 세상의 불균형을 발견하겠습니다"

## 결론

**Capital Imbalance Engine은:**

- 세상의 불균형을 **발견**하는 엔진
- 답을 **생성**하는 엔진이 아님
- 질문을 **개선**하는 엔진
- 증거를 **축적**하는 엔진
- 가설을 **검증**하는 엔진

이 5가지를 할 때, 이 엔진은 **과학**이다.
