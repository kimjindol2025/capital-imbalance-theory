# Capital Observatory — 프로젝트 정체성 선언

**작성일:** 2026-07-03  
**상태:** 최종 정체성 확립

---

## 프로젝트 진화 3단계

### Phase 1: 주식 분석기 (초기)
```
목표: 투자 기회 찾기
초점: 돈 추적
특징: 투자 도구
```

### Phase 2: Flow Engine (중반)
```
목표: 자본 흐름 분석
초점: 불균형 감지
특징: 분석 엔진
```

### Phase 3: Capital Observatory (현재) ✅
```
목표: 불균형 관측
초점: 균형을 잃는 순간 포착
특징: 관측 시스템
```

---

## 핵심 통찰: 관측이 먼저

### 기상청의 본질

```
"기상청은 날씨를 보여주는 곳이 아닙니다.
대기를 관측하는 곳입니다."
```

### Capital Observatory의 본질

```
"Capital Observatory는 자본 흐름을 보여주는 곳이 아닙니다.
불균형을 관측하는 곳입니다."
```

### 구조

```
관측 (Observation) ← 먼저
    ↓
불균형 감지 (Imbalance Detection)
    ↓
자본 이동 (Capital Flow) ← 결과
```

---

## 철학: 불균형이 원인

### Before
```
"오늘 세계 자본이 어디로 흐르나?"

→ 자본 중심
→ 결과 중심
```

### After ✅
```
"오늘 세계에서 가장 큰 불균형은 무엇인가?"
    ↓
"그 불균형 때문에 자본이 어디로 이동하고 있는가?"

→ 불균형 중심
→ 원인 중심
```

---

## 슬로건

### 최종 슬로건

```
"우리는 시장을 예측하지 않는다.
우리는 시장이 균형을 잃는 순간을 가장 먼저 관측한다."
```

### 의미

- **예측 안 함:** 틀릴 가능성 제거
- **관측함:** 투명성 확보
- **가장 먼저:** Lead Time 확보
- **균형을 잃는 순간:** 신호 포착

## 3가지 핵심 개선

### 1️⃣ Observation Process 투명화

#### 구조
```
Evidence (증거) 214개
    ↓
Pressure (압력) Power +3
    ↓
Pattern (패턴) 4개월 연속 증가
    ↓
Observation (관찰) Confidence 61%
```

#### 상호작용
```
사용자가 "Observation" 클릭
    ↓
전체 증거 리스트 표시
    ↓
"아, 이렇게 판단했구나" 신뢰 형성
```

#### 예시
```
Power Pressure +5 (어제 41 → 오늘 46)

원인:
  - 전력 계약 3건 (+2)
  - 데이터센터 2건 (+2)
  - 배터리 공급 1건 (+1)

이 증거들이 모두 보임
```

**효과:** 신뢰도 극대 (맞든 틀리든)

---

### 2️⃣ Daily Journal

#### 목적
```
365일 누적 기록 = 엄청난 자산
```

#### 구조
```
2026-07-03

새로운 Reservoir: ❌ 없음

Changes:
  Power: 41 → 46 (+5)
  AI: -3 → -5 (-2)
  Cooling: 2 → 3 (+1)

Evidence: 5건
Confidence: 61%
```

#### 가치
```
1년 기록 = 신뢰 데이터베이스
→ 5년 기록 = 신뢰 자산
→ 10년 기록 = 금융 인프라
```

**Why Journal > Product:**
- Product: 일회성, UI/UX 개선
- Journal: 장기, 신뢰 누적

---

### 3️⃣ "What Changed Today?" 섹션

#### 구조
```
📊 What Changed Today?

Indicator    어제  오늘   변화
────────────────────────────
Power        41  → 46   (+5)
AI           -3  → -5   (-2)
Cooling       2  →  3   (+1)

원인 분석:
  Power +5:
    - 전력 계약 3건
    - 데이터센터 2건
```

#### 목적
```
"매일 보는 이유"를 제공
→ 신호 포착
→ 변화 추적
→ 습관화
```

## 구조: Product < Journal

### Before
```
Product를 강조
→ 기술 중심
→ "뭐하는 건데?" 느낌
```

### After ✅
```
Journal을 강조
→ 관측 과정 중심
→ "아, 이렇게 알았구나" 신뢰
```

### 우선순위
```
1순위: Journal 품질
       (관측 과정 투명화)

2순위: What Changed
       (일일 신호)

3순위: Product
       (UI/UX 개선)
```

## 지속가능성: 관측의 품질

### Not: 예측 정확도 높이기

```
❌ 예측이 맞으면 신뢰
❌ 예측이 틀리면 신뢰 상실
→ 높은 리스크
```

### But: 관측의 품질 향상

```
✅ 관측이 투명하면 신뢰
✅ 관측이 정확하면 신뢰
✅ 관측 방법이 과학적이면 신뢰
→ 지속가능
```

### 핵심

```
"예측을 늘리는 것보다
관측의 품질을 높이는 것이
이 프로젝트를 오래 살아남게 할 것이다."
```

## 진화의 의미

### 가장 중요한 변화

```
'돈'에서 '불균형'으로 중심축이 이동
```

### Before: 투자 도구
```
돈을 추적한다
→ 투자 기회 찾기
→ 주가 예측
→ 틀릴 수 있음
```

### After: 관측 시스템 ✅
```
불균형을 관측한다
→ 균형을 잃는 순간 포착
→ 투명한 증거 제시
→ 신뢰 누적
```

---

## 메모리: 왜 이것이 중요한가?

### 돈은 결과다

```
불균형 (원인)
    ↓ 물리적 법칙
자본 흐름 (결과)
```

### 우리는 결과를 추적하는 게 아니다

```
❌ "AI에 돈이 많이 가네" (결과 추적)
✅ "AI는 GPU 부족으로 불균형이다" (원인 관측)
```

### 따라서

```
우리가 해야 할 일:
- 불균형을 먼저 관측
- 증거를 투명하게 제시
- 자본 이동을 설명 (결과)
```

## 최종 평가

### 이 프로젝트의 성장

```
처음: "돈을 더 빨리 벌게 해주는 도구"
중간: "자본 흐름을 분석하는 엔진"
지금: "세상이 균형을 잃는 순간을 가장 먼저 관측하는 시스템"

→ 투자 도구 → 관측 시스템으로의 근본적 전환
```

### 이 전환의 가치

```
투자 도구 = 기술 스타트업
관측 시스템 = 금융 인프라

전자는 떠난다.
후자는 남는다.
```

## 앞으로의 방향

### 개선할 영역

1. **Observation Process**
   - 증거 → 압력 → 패턴 → 관찰 (투명화)

2. **Daily Journal**
   - 365일 누적 기록 (신뢰 자산 구축)

3. **What Changed**
   - 매일 보는 이유 제공 (습관화)

### 피해야 할 영역

1. **예측 모델 추가** ❌
   - 정확도 게임으로 빠질 수 있음

2. **복잡한 분석** ❌
   - 투명성 손실

3. **UI/UX 과다 개선** ❌
   - 기술 중심이 될 수 있음

### 집중해야 할 것

1. **관측의 품질 높이기** ✅
   - 증거 수집 방법 개선
   - Confidence 메트릭 정교화

2. **Journal 기록 확실히 하기** ✅
   - 365일 누적
   - 검색/분석 기능 강화

3. **신호 포착 정확도** ✅
   - 새 Reservoir 감지
   - False positive 감소
