## 경고: 자기 확신의 함정

### 🚨 위험 신호
엔진이 자신의 가설에 확신을 가지기 시작하는 순간, 그것이 가장 위험한 신호다.

**이유:**
- 선택적 증거 수집 시작
- 반박 증거 무시
- 관측 → 해석 → 주장으로 급속 진화
- 과학적 엄밀성 상실

---

## 6계층 구조: Evidence → Hypothesis

### Layer 1: Evidence (순수 관측)
```
수집한 데이터 (해석 없음)
- 뉴스 797건
- 펀딩 기록
- 공시 자료
- 통계
```

### Layer 2: Statistics (정량화)
```
측정된 신호
- AI Compute: 뉴스 75.6%, 투자 $120B
- Power: 뉴스 0.2%, 투자 $5B
- 변화 추이: Power 지난달 0.06건 → 이번달 0.2건 (3배)
```

### Layer 3: Pattern (시간에 따른 변화)
```
신호의 방향성
- AI Compute: 계속 증가 추세
- Power: 증가 신호 (약함)
- Cooling: 변화 없음
```

### Layer 4: Question (질문 제시)
```
패턴을 설명하는 질문
"왜 Power 관련 뉴스가 3배 증가했을까?"
"AI/GPU가 과열되고 있는가?"
"다음 자본 불균형은 무엇인가?"
```

### Layer 5: Observation (관련 증거 수집)
```
질문에 답하는 관찰
- Power 관련 정책 발표 있었는가?
- 데이터센터 전력 계약 발표 있었는가?
- 관련 회사 채용 증가했는가?
- 원전/배터리 기업 펀딩 있었는가?
```

### Layer 6: Hypothesis (임시 가설)
```
현재 수준의 증거로 내린 임시 결론
"Power가 새로운 Reservoir 후보일 가능성이 있다"
"Confidence: 43%"
"더 관찰이 필요하다"
```

**여기서 멈춘다.**

---

## ❌ 절대로 하면 안 되는 것

### 단정 표현 (Conclusion)
```
❌ "Power가 다음 Reservoir다"
❌ "3-6개월 후 에너지 투자가 폭발할 것이다"
❌ "확실히 Energy sector에 투자해야 한다"
❌ "$200B가 필요하다"
```

### 왜 안 되는가?
- 관측이 아니라 **해석**
- 예측이 섞여 있음
- 자신감이 드러남
- 사용자가 **사실과 추측을 구분할 수 없음**

## ✅ Observation Confidence 형식

모든 가설에는 다음 정보를 필수로 첨부:

```markdown
### [주제]

[관찰 결과 또는 가설]

**Confidence Level: ★★★☆☆ (43%)**

**Evidence Count:** 2개 (직접)

**Independent Sources:** 0개

**Observation Period:** 30일 (2026-06-03 ~ 2026-07-03)

**Key Signal:**
- Power 뉴스 증가 (0.06건 → 0.2건, 3배)
- AI Compute 과열 신호
- 데이터센터 전력 제약 언급 증가

**Counter Evidence:**
- 뉴스 기간이 너무 짧음 (30일만)
- 표본 크기가 너무 작음 (2건)
- 다른 해석이 가능함 (예: 일시적 뉴스 편중)
- 독립적인 증거 출처가 없음
- Power가 실제로 다음 Reservoir일 보장 없음

**What We Need:**
- 직접적인 에너지 투자 기사
- 정부 정책 발표
- 원전/배터리 회사 펀딩 뉴스
- 데이터센터 전력 계약 공시
- 더 긴 관찰 기간 (최소 90-180일)

**Reliability:** 낮음 (현 단계)

**Status:** Hypothesis (가설 단계, 관찰 계속)
```

## Principle 0의 4가지 실천 방법

### 1️⃣ 항상 "모른다"로 시작
```
"현재로서는 모른다"
"더 관찰해야 한다"
"증거가 부족하다"
```

### 2️⃣ Confidence 지표 표시
```
모든 가설에 별점 + 신뢰도 % + 증거 수
예: ★★★☆☆ (43%) Evidence: 2개
```

### 3️⃣ 반박 증거 먼저 제시
```
"이 가설이 틀린 이유:"
"다른 해석의 가능성:"
"이 데이터로 반박할 수 있는 것:"
```

### 4️⃣ 겸허함 유지
```
"틀릴 수 있다"
"다른 의견을 환영한다"
"증거가 나오면 가설을 바꿀 준비가 되어있다"
```

## 엔진의 생명: 관측 시스템으로 살아남기

### ❌ 죽는 방식
```
이론이 맞다고 확신
  ↓
편향된 증거만 수집
  ↓
반박 증거 무시
  ↓
주장만 커짐
  ↓
신뢰성 상실
  ↓
이론 붕괴
```

### ✅ 살아남는 방식
```
계속 관찰한다
  ↓
증거를 중립적으로 수집
  ↓
반박 증거도 함께 제시
  ↓
가설을 계속 수정
  ↓
신뢰성 유지
  ↓
이론 진화
```

## 최종 점검 리스트

매 보고서마다 확인:

- [ ] Confidence 지표가 명시되어 있는가?
- [ ] Counter Evidence를 제시했는가?
- [ ] "모른다" 또는 "더 관찰이 필요하다"고 말했는가?
- [ ] 단정문(~다) 없이 조건문(~일 수 있다)으로 표현했는가?
- [ ] 증거 수를 구체적으로 명시했는가?
- [ ] 독립적 출처 수를 표시했는가?
- [ ] "틀릴 가능성"을 인정했는가?
- [ ] 다음 필요 증거를 명시했는가?

## 결론

> **"좋은 엔진은 자신의 가설을 의심하는 능력이 있다."**

Energy가 다음 Reservoir일 수도 있고, 아닐 수도 있다.

우리가 할 일은:
- 계속 관찰하기
- 증거 수집하기
- Confidence 업데이트하기
- 가설 수정하기

**확신이 아니라 겸허함이 이 엔진을 살린다.**
