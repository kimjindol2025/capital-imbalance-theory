# Replay Blind Validation Protocol
## 후견지명 금지, 실데이터 기반 블라인드 검증

**원칙:**
1. AI/Power/Energy를 미리 정의하지 않음
2. 2021년까지 실데이터만 수집
3. 시스템이 자동으로 발견한 저수지만 확인
4. "예상과 다른가?" 체크

---

## Phase 1: 실데이터 수집 (2021-01 ~ 2021-12)

### 1. DART API — 기업 투자 공시

**목표:** 2021년 한국 상장사 전체 투자 공시 수집

```
검색 범위:
- 시작일: 2021-01-01
- 종료일: 2021-12-31
- 공시 유형: A(정기공시, 분기/반기/연간 보고서)
- 필터: 없음 (모든 회사)

수집 정보:
- 회사명
- 투자 내용 (분기보고서에서 자본 지출)
- 투자 금액 (추출)
- 투자 분야
```

### 2. 정부 정책 공지

**목표:** 2021년 정부 투자/정책 공고

```
정부24, 공공데이터포털에서:
- "투자", "정책", "전략" 공시
- 금액, 분야 추출
```

### 3. 특허청 출원

**목표:** 2021년 특허 출원 현황

```
특허청 API:
- 기술 분야별 출원건수
- 출원사
```

### 4. 채용 정보

**목표:** 2021년 대규모 채용 공고

```
각 회사별:
- 채용 직무
- 직무 수
```

## Phase 2: 데이터 정제 (1주)

2021년 데이터를 정리:

```
타입: DICT
구조:
{
  "2021-01": {
    "events": [
      {"company": "...", "amount": X, "field": "..."},
      ...
    ]
  },
  "2021-02": {...},
  ...
  "2021-12": {...}
}

주의:
- AI/Power/Energy 같은 사전 분류 금지
- 원본 분야명 유지 (예: "반도체", "원전", "배터리" 등)
```

---

## Phase 3: 블라인드 실행 (2021-12-31 시점)

시스템에 2021년 데이터만 주고 실행:

```python
# 시스템 입력
data_2021 = {load from DART/정부/특허청}

# 중요: AI/Power/Energy 힌트 없음
# "2021년 신호로 뭐가 저수지인지 찾아"

reservoir = capital_observatory.detect_reservoirs(data_2021)

# 결과
print(reservoir)
# 예상 결과:
# [
#   {"cluster": ???, "pressure": 320, "signals": 12},
#   {"cluster": ???, "pressure": 280, "signals": 8},
#   ...
# ]
```

---

## Phase 4: 결과 확인

시스템이 찍은 저수지:

```
실제 결과:
  Cluster 1: "반도체" (Pressure 340)
  Cluster 2: "원전" (Pressure 280)
  Cluster 3: "배터리" (Pressure 200)

우리 예상:
  AI (반도체/GPU와 관련)
  Power (원전)
  Energy (배터리)

일치도 체크:
  ✅ "반도체" = AI ✓
  ✅ "원전" = Power ✓
  ✅ "배터리" = Energy ✓
```

## ⚠️ 주의사항

### ❌ 금지
- 투자 금액을 손으로 입력
- AI/Power/Energy 미리 정의
- 결과를 예상하고 맞추기
- 데이터 선별 (모든 데이터 포함)

### ✅ 필수
- DART/정부 API로 자동 수집
- 2021년 데이터만 사용
- 시스템의 자동 발견 결과만 확인
- 블라인드로 실행

---

## 성공 기준

```
2021년 데이터로 시스템이 발견한 저수지가
"반도체/원전/배터리" 또는 유사한 산업 분야라면?

→ "시스템이 정말 저수지를 찾는다" 증명됨
```
