# Capital Observatory — Sensor Observatory Manifesto

**선언일:** 2026-07-03  
**상태:** 프로젝트 근본적 재설계 완료

---

## 가장 중요한 한 줄

> **"우리는 뉴스를 수집하지 않는다. 우리는 세상을 관측하는 센서들의 신호를 수집한다."**

이 순간부터 프로젝트는 **뉴스 엔진**이 아니라 **관측 시스템**으로 완전히 성격이 바뀝니다.

---

## 가장 큰 오류

```
❌ 뉴스를 중심으로 생각했다

"뉴스를 모으면 된다" → 검색 → 편향

✅ 센서를 중심으로 생각한다

"세상의 신호를 모은다" → Crawl → 객관성
```

---

## 신 아키텍처

### Layer 0: Sensor (입력층)

세상에서 흘러나오는 신호들:

```
📰 뉴스           대중의 관심도
📋 공시           기업의 의도 (Fact)
🏛️ 정부           정책 (Fact)
🛒 입찰           구매 신호
🔬 특허           기술 진전
👥 채용           인력 확충
🏢 기업 IR        경영진 계획
💰 가격/주가      시장 판단
📊 수출입         무역 흐름
⏱️ 리드타임       물리적 제약
```

**뉴스는 10개 센서 중 1개일 뿐입니다.**

### Layer 1: Event Detector (필터층)

각 신호에서:
```
질문: "이게 Flow Event인가?"

YES → 저장
NO  → 폐기
```

예:
```
삼성 20조 투자        YES (공시)
CEO 인터뷰            NO  (뉴스)
특허 출원 5배 증가     YES (특허)
채용 공고 3배         YES (채용)
기사 댓글 증가        NO  (SNS)
```

### Layer 2: Event Importance (등급층)

Flow Event 중요도 등급:

```
국가 정책           ★★★★★ (100)
기업 공시           ★★★★☆ (90)
장비 발주           ★★★★☆ (80)
특허 출원           ★★★☆☆ (60)
채용 공고           ★★☆☆☆ (40)
언론 인터뷰         ★☆☆☆☆ (10)
SNS 감정            ★☆☆☆☆ (5)
```

### Layer 3: Reservoir Candidate (분석층)

등급화된 Event들로:
- Pressure 계산
- Reservoir 감지
- 불균형 관측

## 센서 우선순위 (중요도 역순)

### 1️⃣ 공시 (Corporate Fact)
```
기업의 공식 발표
가장 신뢰도 높음
시간 정확
정량화 가능
```

### 2️⃣ 정부 (Government)
```
정책, 법안, 입찰
공식 결정
영향 크다
```

### 3️⃣ 입찰 (Procurement)
```
구체적 구매 의사
자본 흐름 신호
```

### 4️⃣ 특허 (Patent)
```
기술 진전 객관적 증거
경쟁사 분석 가능
```

### 5️⃣ 채용 (Recruitment)
```
인력 확충 = 자본 투입
성장 신호
```

### 6️⃣ 기업 IR (Investor Relations)
```
경영진 계획 발표
향후 방향성
```

### 7️⃣ 뉴스 (News) ⚠️
```
이미 누군가 가공한 정보
원본이 아님
시간 지연 (1-3일)
주관 포함 가능
검색 시 편향 불가피

⚠️ 뉴스가 7번째입니다!
```

### 8️⃣ SNS (Social Media)
```
감정과 추측
신뢰도 낮음
noise 많음
```

## 왜 뉴스가 마지막인가?

### 뉴스의 근본적 문제

```
1. 이미 가공된 정보
   원본이 아니고 편집됨
   
2. 시간 지연
   공시 → 뉴스까지 1-3일
   
3. 주관 개입
   기자 해석 포함
   
4. 검색 불가피
   어떤 뉴스를 찾을지 사람이 결정
   → 편향 피할 수 없음
   
5. 트렌드에 민감
   AI가 유행이면 AI만 보도
   Power는 관심 없음
```

### 공시의 장점

```
1. Fact
   기업이 공식 발표한 내용
   
2. 객관성
   주관 개입 불가능
   
3. 시간 정확
   정확한 순서대로 기록
   
4. 정량화 가능
   투자액, 수량, 일정 등
   
5. 검증 가능
   거짓말하면 처벌됨
```

## Search vs Crawl

### ❌ Search (피해야 함)

```
사람이 "무엇을 찾을지" 결정
    ↓
"AI+ChatGPT" 검색
    ↓
AI만 많은 결과
    ↓
편향 불가피
```

### ✅ Crawl (해야 함)

```
세상의 모든 신호 수집
    ↓
AI가 "Flow Event"만 필터링
    ↓
사람의 의도 제거
    ↓
객관성 최대화
```

---

## Observatory vs Search Engine

### Search Engine (❌ 우리가 아님)
- 사람: "AI 검색" → AI 결과만 → 편향
- 정체: 선택 필터
- 문제: 확인 편향

### Observatory (✅ 우리의 정체)
- 천문대처럼 하늘을 계속 봄
- 세상의 모든 신호를 지속 수집
- AI가 필요한 신호만 추출
- 사람의 의도 제거

## 프로젝트 최종 재정의

### Before: "뉴스 엔진"
```
뉴스 중심
검색 기반 (Search)
편향 불가피
기자 같은 시스템
```

### After: "센서 관측 시스템" ✅
```
센서 중심 (10개 신호)
Crawl 기반 (전수 수집)
편향 최소화 (필터 기반)
천문학자 같은 시스템 (Observatory)
```

---

## 질문의 변화

### Before
```
"뉴스에서 뭘 찾을까?"
→ Search
→ 편향
```

### After ✅
```
"세상의 신호에서 뭘 봤을까?"
→ Observe
→ 객관성
```

## 최종 철학

```
센서들이 같은 방향을 가리킬 때
비로소 Pressure가 생긴다.

그 Pressure가 모이는 곳이
Reservoir다.

우리는 그 순간을
가장 먼저 관측한다.

우리는 뉴스를 수집하지 않는다.
우리는 세상을 관측하는 센서들의 신호를 수집한다.
```
