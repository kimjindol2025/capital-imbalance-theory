## 📚 영상의 흐름

### 1️⃣ **문제 제시: 학습 지옥의 순환**

```
현재 상태:
├─ 브라우저 탭: 강의 영상 12개 (입문, 기초, 도구 등)
├─ 노션: "Vibe Coding 학습 로드맵" (웅장한 계획)
├─ 채팅방: 다른 사람의 성공 사례 (부러움)
└─ 실제 결과: 만든 프로그램 0개

시간 경과:
날 1: 강의 결제 + 도구 설치
일주일 후: 강의 1개 더 결제 + 새로운 도구 추가
1개월 후: 준비만 2번째 버전... 만든 것은 여전히 0개
```

**질문:** 시간이 부족해서? 기초가 부족해서? 더 나은 도구가 필요해서?

### 2️⃣ **근본 원인: 숨겨진 목표**

덴코(Denko)의 주장:
> **결과를 바꾸고 싶다면 결과를 만들어내는 행동부터 먼저 바꿔야 한다**

#### 행동과 말의 불일치

```
말: "프로그램을 만들고 싶다"
실제 행동: "학습하는 척하며 실패를 피하고 싶다"
```

#### 진짜 목표는?

- 강의만 듣는 사람 → **학습 완료의 쾌감**
- 계속 프롬프트 수집 → **도구 탓할 거리 확보**
- 노션에 계획 작성 → **준비 중이라는 안전감**

---

### 3️⃣ **정체성이 가장 큰 장벽**

```
"나는 비전공자다"
↓ (오래 반복하면)
↓
"나는 영원히 만들 수 없는 사람이다"
```

#### 정체성의 힘

- **현재를 설명:** "비전공자라서 지금은 못 만들어"
- **미래를 통제:** 터미널 화면만 나와도 경기 일으킴
- **책임 회피:** 오류 = "나는 이 일을 하면 안 되는 사람의 증거"

**중요한 통찰:**
> 오류는 신분을 확인하지 않는다.  
> 전공자에게도 평등하다.  
> 개발자는 오류가 없는 사람이 아니라  
> 오류를 보고 다음 질문을 만드는 사람이다.

---

### 4️⃣ **Loop Engineering (바이브 코딩의 핵심)**

```
Step 1: 프로그램 아이디어
    ↓
Step 2: AI에게 요청
    ↓
Step 3: 결과 실행해보기
    ↓
Step 4: 현재 vs 원하는 상태 비교
    ↓
Step 5: 차이점 발견 → 다시 요청
    ↓
Step 1 (반복)
```

#### 비전공자의 실수

```
❌ "AI가 한 번에 못 만들었으니 나는 재능이 없다"
   = "전자레인지가 덜 데워졌으니 요리 재능이 없다" (?)

✅ "시간을 조금 더 돌리면 된다"
   버튼 위치를 옮기고, 저장 오류를 찾고, 화면을 정리한다
```

---

### 5️⃣ **AI는 구현을 도와주지만, 욕망은 본인이 정해야 한다**

```
AI가 할 수 있는 것:
├─ 코드 작성
├─ 기능 구현
├─ 오류 수정
└─ 아이디어 20개 제안

AI가 할 수 없는 것:
├─ "너는 뭐가 짜증나니?"
├─ "어떤 업무를 없애고 싶니?"
├─ "무엇을 직접 통제하고 싶니?"
└─ 욕망 결정 ⭐
```

---

### 6️⃣ **작은 것부터 시작하기 - 게임 방식**

#### 구조

```
비전 (1년): 필요한 도구 12개 완성

목표 (1개월): 그 중 1개 완성

퀘스트 (오늘): 로그인 화면 1개 or 오류 1개 해결
```

#### 구체적 예시

```
슬라임 처치 (쉬운 것부터):
├─ 내가 사용할 계산기
├─ 강의 신청자 정리 화면
├─ 반복 문장 자동 작성 스킬
├─ 파일 이름 일괄 변경 스킬
└─ → 이런 것이 "행동의 증거"가 된다
```

## 💡 주요 학습 포인트

### 1. 정체성 vs 현실

```
"나는 비전공자다" → (사실)
"그래서 영원히 못 만든다" → (거짓)

현재의 경험 ≠ 미래의 가능성
```

### 2. 오류는 신호

```
❌ 오류 = 나는 못하는 사람
✅ 오류 = 이제 고칠 게 생겼다
```

### 3. Loop의 중요성

```
완벽한 첫 요청 → 0%
만들고 → 보고 → 수정하고 → 반복 → 100%
```

### 4. 욕망을 먼저 정해야 함

```
AI: "뭘 만들까요?" (20가지 제안)
당신: "아니야, 내가 짜증나는 건..." (1가지 명확함)
```

### 5. 작은 도구의 힘

```
큰 목표 1개 (계획)
└─ 작은 도구 12개 (현실화)
    ├─ 계산기
    ├─ 정리 화면
    ├─ 자동화 스킬
    └─ ...
```
